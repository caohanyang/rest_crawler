Place Guest Order

POST /guest_checkout_session/{checkoutSessionId}/place_order

This call pays for the items and terminates the specified guest checkout session.

Also see Negative Testing Using Stubs for information on how to emulate error conditions for this call using stubs.

Note: The X-EBAY-C-ENDUSERCTX header is required when submitting this call. This header specifies the risk correlation ID, which is provided by a security software library.

For example:
X-EBAY-C-ENDUSERCTX deviceId=< riskCorrelationId >

In addition to the risk correlation ID, the following settings are used for affiliate purchase transactions. These settings provide a way for affiliates to track purchases at the user level. This data can then be used in their campaigns to grow user loyalty.

  • affilliateCampaignId - The identifier of the affiliate's campaign, which is obtained from the eBay Partner Network. Required for revenue sharing for guest checkout transactions.
  • affilliateReferenceId - Any value you want to use to identify this transaction in the Affiliate reports. Optional

purchaseOrderId string Conditionally A unique identifier of the purchase order. When a checkout session completes, a purchase order Id is generated.
purchaseOrderPaymentStatus string Conditionally Indicates the payment status for the purchase order.

Applicable values are from PurchaseOrderPaymentStatusEnum :

FAILED
Indicates that processing the payment for the purchase order has failed.
PAID
Indicates the purchase order has been paid for.
PARTIALLY_PAID
Indicates that only some of the line items in the purchase order have been paid for.
PENDING
Indicates the payment for the purchase order is pending.

Error Codes

Code Domain Category Nature Meaning
15000 API_ORDER APPLICATION ERROR There was a problem with an eBay internal system or process. Contact eBay developer support for assistance.
15003 API_ORDER REQUEST ERROR The checkout session requested does not exist.
15004 API_ORDER REQUEST ERROR If this is a eBay member checkout, this error indicates that the buyer does not have a PayPal account linked to their eBay account. If this is a guest checkout, this indicates that the buyer's credit card information is missing. In both cases, submit the buyer's credit card information.
15005 API_ORDER BUSINESS WARNING Some of the line items were not purchased. Use the Order API getPurchaseOrder call to check the payment status of each line item.
15006 API_ORDER BUSINESS WARNING The order has already been placed for the checkout session submitted and no modifications can be made. Use the Order API getPurchaseOrder call to get the purchase order details.
15019 API_ORDER BUSINESS ERROR To place an order, you must have at least one line item. Use the initiateCheckoutSession call to add line items (maximum of 4) and create a new checkout session.
15020 API_ORDER BUSINESS ERROR During the checkout process the item has been changed. Create a new checkout session for this item using the initiateCheckoutSession call.
15023 API_ORDER BUSINESS ERROR The credit card was declined. Use the updatePaymentInfo call to change the payment information.
15024 API_ORDER BUSINESS ERROR There is a problem with the buyer's PayPal account. The buyer should check their account or provide a credit card to pay for the order.
15025 API_ORDER REQUEST ERROR App is not authorized to access this resource.

Sample: Place the Order

Creates the purchase order and starts the process that pays for the line items.

Description

This call generates the purchase order ID and starts the process that pays for the line items for a guest checkout. Be sure to store this ID because it is passed as a URI parameter in the Get Guest Purchase Order call.

Input

The input is the checkoutSessionId. 

URL format. See also the non-wrapped version of this URL.

POST https://api.ebay.com/buy/order/v1/guest_checkout_session/v1|5018578757|63iqkv_LQQy/place_order

Output

The output is the purchase order ID. 

JSON format.
{
  "purchaseOrderId": "140000000577983",
  "purchaseOrderHref": "http://orderapi.ebay.com/buy/order/v1/guest_purchase_order/140000000577983",
  "purchaseOrderPaymentStatus": "PAID"
}
