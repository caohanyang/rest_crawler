InventoryItem

This type is used to provide detailed information about an inventory item.

Type that uses InventoryItem:

    InventoryItems

Calls that use InventoryItem:

    /inventory_item GET , /inventory_item/{sku} PUT

Fields

availability ( Availability )
condition ( ConditionEnum )
conditionDescription ( string )
packageWeightAndSize ( PackageWeightAndSize )
product ( Product )
sku ( string )


availability ( Availability ) [0..?]
This container is used to specify the quantity of the inventory item that are available for purchase if the item will be shipped to the buyer via a standard shipping method.

Note: The pickupAtLocationAvailability container is not yet available for use. In the future, this container will allow sellers to set the quantity of the inventory item that are available for In-Store Pickup or Click and Collect at one or more of the merchant's physical stores.
See the Field Index to learn which calls use availability.

condition ( ConditionEnum ) [0..?]
This enumeration value indicates the condition of the inventory item. See ConditionEnum for valid condition values for this field.
See the Field Index to learn which calls use condition.

conditionDescription ( string ) [0..?]
This string field is used by the seller to more clearly describe the condition of used inventory items, or inventory items whose condition value is not NEW , LIKE_NEW , NEW_OTHER , or NEW_WITH_DEFECTS

The ConditionDescription field is available for all eBay categories. If the conditionDescription field is used with an item in the new condition (mentioned in previous paragraph), eBay will simply ignore this field if included, and eBay will return a warning message to the user.

This field should only be used to further clarify the condition of the used item. It should not be used for branding, promotions, shipping, returns, payment or other information unrelated to the condition of the item. Make sure that the condition value, condition description, listing description, and the item's pictures do not contradict one another.
.
See the Field Index to learn which calls use conditionDescription.

packageWeightAndSize ( PackageWeightAndSize ) [0..?]
Required if the seller is offering one or more calculated shipping options, or if the seller is offering flat-rate shipping but is including a shipping surcharge based on the item's weight. This container is used to specify the dimensions and weight of a shipping package.
See the Field Index to learn which calls use packageWeightAndSize.

product ( Product ) [0..?]
This container is used to define the product details, such as a title, a product description, item specifics, and links to images for the product. Optionally, in a Create or Replace Inventory Item call, a seller can pass in a Global Trade Identification Number (GTIN) value such as an EAN, an ISBN, a UPC, or a Brand and Manufacturer Part Number (MPN) pair to identify a product to be matched with a product in the eBay catalog. If a match is found in the eBay product catalog, the inventory item is automatically populated with available product details, such as the title, the product description, item specifics, and a link to the stock image of the product.
See the Field Index to learn which calls use product.

sku ( string ) [0..?]
The seller-defined Stock-Keeping Unit (SKU) of the inventory item. The seller must have a unique SKU value for every product that they sell on eBay.

Note: This field is only applicable for the two "Get" calls, and is not used by the Create or Replace Inventory Item call. If a sku field is passed in to the request payload of a Create or Replace Inventory Item call, it will be ignored. For a Create or Replace Inventory Item call, the SKU value is passed in as part of the call URI.
See the Field Index to learn which calls use sku.
