Search for Items

GET /item_summary/search

This call is used to perform an advanced search for items. In addition to the required keyword, users can search by category and use filters to limit the results by listing format, item condition, price range, UPC value, listing end date, location, and more. Multiple filter types can be specified in one call. There are also pagination controls and a sort query parameter to control/sort the data that is returned.

Input

See also Samples .

Resource URI (production) 

GET https://api.ebay.com/buy/browse/v1/item_summary/search?
  category_ids=string&
  filter=FilterField&
  limit=string&
  offset=string&
  q=string&
  sort=SortField

URI parameters

Parameter Type Required? Meaning
category_ids string Optional This query parameter is used to retrieve items in a specific eBay category. Below is the syntax for using this query parameter.

GET https://api.ebay.com/buy/browse/v1/item_summary/search?category_ids=29792

Note: If a top-level (L1) category is specified, you must also include the q query parameter. Below is an example of this use case:
GET https://api.ebay.com/buy/browse/v1/item_summary/search?q=Outliers&category;_ids=1854946


filter string Optional This query parameter supports multiple field filters that can be used to use to limit/customize the result set. See Search Filters Usage for information about each of these filters.

Applicable values are from FilterField .
limit string Optional This value is used to specify the maximum number of items to display on a single 'page' of data. This value defaults to 50 if the limit parameter is omitted.

Default: 50
Maximum: 200


offset string Optional This value sets the specific 'page' of data to return. This value should be set to 0 to return the first page of data, 1 to return the second page of data, and so on. Based on the number of result pages that exist, it is possible to set this value too high. This value defaults to 0 (first page of data is returned) if the offset parameter is omitted.

Default: 0

q string Required A string consisting of one or more words that are used for a search of items on eBay.

For example: "iphone" or "Harry Potter".

When keywords are separated by a comma, the comma is ignored. So a keyword of "iphone, ipad" is the same as using "iphone ipad". Both return items that have iphone or ipad.



sort string Optional Specifies the order and the field name to use to sort the items. The default order is ascending. To sort in descending order use - before the field name. Currently, users can only sort by price (in ascending or descending order), or by distance (only applicable if the local pickup filters are used, and only ascending order is supported).

The following are examples of using the sort query parameter.

Sort Result
&sort;=price sorts by price in ascending order (lowest price first)
&sort;=-price sorts by price in descending order (highest price first)
&sort;=distance sorts by distance in ascending order (shortest distance first)

Default: by price (ascending)

Applicable values are from SortField .
