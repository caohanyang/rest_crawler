Get all Seller Profiles

GET /seller_standards_profile

This call retrieves all the profiles for the associated seller.

Input Output Samples Change History

Input

See also Samples .

Resource URI (production) 

GET https://api.ebay.com/sell/analytics/v1/seller_standards_profile

This call has no path or query parameters.


HTTP request headers

All requests made to eBay REST operations require you to provide the authorization HTTP header for authentication.
See HTTP request headers for details.



OAuth request scope

This request requires a user access token with the following scope:

https://api.ebay.com/oauth/api_scope/sell.analytics.readonly

See Getting Access Tokens for more information.



Payload model

This call has no request payload.

Response field descriptions



Output Container/Field Type Occurrence Meaning
standardsProfiles array of StandardsProfile Conditionally A list standards profiles for the seller. A standardsProfile is a combination of cycle and a program.
standardsProfiles. cycle Cycle Conditionally Indicates the cycle of the evaluation, whether CURRENT or PROJECTED. Each program will have at least one cycle, but a program can include both cycles.
standardsProfiles.cycle
. cycleType 		string Conditionally The cycle type, either CURRENT or PROJECTED.

Applicable values are from CycleTypeEnum :

CURRENT
Indicates the evaluation that determines your current, effective seller level.
PROJECTED
Indicates the seller level that would take effect on the next evaluation.

Code so that your app gracefully handles any future changes to this list.
standardsProfiles.cycle
. evaluationDate 		string Conditionally The date on which the current, effective seller level was computed.
standardsProfiles.cycle
. evaluationMonth 		string Conditionally The month in which the current, effective seller level was computed.
standardsProfiles
. defaultProgram 		boolean Conditionally If set to true , it indicates this is the default program.

Except for sellers in China, a seller's default program is the site where the seller registered with eBay. Seller's in China select their default program when they register.

standardsProfiles
. evaluationReason 		string Conditionally The type of logic used to calculate your overall seller level. eBay may override your calculated seller level if eBay determines there are special circumstances that warrant an override. In general, overrides performed by eBay protect your seller level.
standardsProfiles. metrics array of Metric Conditionally A list of the metrics that contributed to the evaluation. See the applicable metrics and requirements for each program:

eBay Top Rated seller program standards

standardsProfiles.metrics
. level 		string Conditionally Indicates the seller level for this metric.

Applicable values are from StandardsLevelEnum :

ABOVE_STANDARD
Indicates the seller or metric meets the Above Standard requirement.
BELOW_STANDARD
Indicates the seller or metric does not meet the minimum standards performance requirement.
TOP_RATED
Indicates the seller or metric meets the Top Rated requirement.

Code so that your app gracefully handles any future changes to this list.
standardsProfiles.metrics
. lookbackEndDate 		string Conditionally The end date of the period during which the seller was evaluated for the associated metric.
standardsProfiles.metrics
. lookbackStartDate 		string Conditionally The beginning date of the period during which the seller was evaluated for the associated metric.
standardsProfiles.metrics
. metricKey 		string Conditionally The name of the metric. For example, Defect rate .
standardsProfiles.metrics. name string Conditionally The complete name of the metric for which the seller is being evaluated. This value is localized according to the value of the X-EBAY-C-MARKETPLACE-ID request header.
standardsProfiles.metrics
. thresholdLowerBound 		any Conditionally The lower limit of the metric threshold used in the evaluation. Each program can have different threshold levels.
standardsProfiles.metrics
. thresholdMetaData 		string Conditionally Expression for the type of metric, and the inclusive and exclusive characteristics of the upper and lower bounds of the threshold. Thresholds are shown for either a count or a rate.
For example, the following metadata means that to maintain the level for this metric, you must continue to have a value of 90 or better, and there is no upper bound for this metric:
[90, NA)
Metadata values are enclosed by parentheses and/or square brackets. A parenthesis means the adjacent value is exclusive, while a square bracket means the adjacent value is inclusive.
standardsProfiles.metrics
. thresholdUpperBound 		any Conditionally The upper limit of the metric threshold used in the evaluation. Each program can have different threshold levels.
standardsProfiles.metrics. type string Conditionally Indicates the data type of the returned metric, such as a string or integer value.

Applicable values are from DataTypeEnum :See type .
Code so that your app gracefully handles any future changes to this list.
standardsProfiles.metrics
. value 		any Conditionally The seller's value, or score, for the metric.
standardsProfiles. program string Conditionally Indicates the program to which the profile belongs.

Applicable values are from ProgramEnum :

PROGRAM_DE
Indicates the program region is Germany, which includes DE (Germany), AT (Austria), and CH (Switzerland).
PROGRAM_GLOBAL
Includes all defined programs, plus any marketplace countries where a seller has conducted business.
PROGRAM_UK
Indicates the program region is the United Kingdom, which includes both UK (United Kingdom) and IE (Ireland).
PROGRAM_US
Indicates the program region is the United States, which includes US (the United States) and US MOTORS.

Code so that your app gracefully handles any future changes to this list.
standardsProfiles
. standardslevel 		string Conditionally Indicates the overall standards level of the seller.

Applicable values are from StandardsLevelEnum :

ABOVE_STANDARD
Indicates the seller or metric meets the Above Standard requirement.
BELOW_STANDARD
Indicates the seller or metric does not meet the minimum standards performance requirement.
TOP_RATED
Indicates the seller or metric meets the Top Rated requirement.

Code so that your app gracefully handles any future changes to this list.
null
