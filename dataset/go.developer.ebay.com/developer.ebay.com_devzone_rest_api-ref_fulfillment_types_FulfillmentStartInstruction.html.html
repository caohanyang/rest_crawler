FulfillmentStartInstruction

This type contains a set of specifications for processing a fulfillment of an order, including the type of fulfillment, shipping carrier and service, addressing details, and estimated delivery window. These instructions are derived from the buyer's and seller's eBay account preferences, the listing parameters, and the buyer's checkout selections. The seller can use them as a starting point for packaging, addressing, and shipping the order.

Type that uses FulfillmentStartInstruction:

    Order

Calls that use FulfillmentStartInstruction:

    /order GET , /order/{orderId} GET

Fields

ebaySupportedFulfillment ( boolean ) [0..?]
A value of true indicates that the value of the fulfillmentInstructionsType field is SHIP_TO , and that the fulfillment will be shipped via eBay's Global Shipping Program.

For more information, see Managing Shipping .
See the Field Index to learn which calls use ebaySupportedFulfillment.

finalDestinationAddress ( Address ) [0..?]
Contains the final destination address of the shipment.

This address is returned only for Global Shipping Program shipments. Sellers should not ship directly to this address; instead they should ship this package to their international shipping provider's domestic warehouse. The international shipping provider is responsible for delivery to the final destination address.

For more information, see Addressing a Global Shipping Program Shipment .
See the Field Index to learn which calls use finalDestinationAddress.

fulfillmentInstructionsType ( FulfillmentInstructionsType ) [0..?]
The method of fulfillment that will be used to deliver this set of line items to the buyer.
See the Field Index to learn which calls use fulfillmentInstructionsType.

maxEstimatedDeliveryDate ( string ) [0..?]
NOT returned if the value of the fulfillmentInstructionsType field is DIGITAL or PREPARE_FOR_PICKUP , or if the order has promotional free shipping. This is the estimated latest date that the fulfillment will be completed. The timestamp is in ISO 8601 format, which uses the 24-hour Universal Coordinated Time (UTC) clock.

Format: YYYY-MM-DDTHH:MM:SS.SSSZ
Example: 2015-08-04T19:09:02.768Z
See the Field Index to learn which calls use maxEstimatedDeliveryDate.

minEstimatedDeliveryDate ( string ) [0..?]
NOT returned if the value of the fulfillmentInstructionsType field is DIGITAL or PREPARE_FOR_PICKUP , or if the order has promotional free shipping. This is the estimated earliest date that the fulfillment will be completed. The timestamp is in ISO 8601 format, which uses the 24-hour Universal Coordinated Time (UTC) clock.

Format: YYYY-MM-DDTHH:MM:SS.SSSZ
Example: 2015-08-04T19:09:02.768Z
See the Field Index to learn which calls use minEstimatedDeliveryDate.

shippingStep ( ShippingStep ) [0..?]
Returned only if the value of fulfillmentInstructionsType is SHIP_TO . Contains shipping information for this fulfillment, including the carrier, the service, and the shipment destination.
See the Field Index to learn which calls use shippingStep.
