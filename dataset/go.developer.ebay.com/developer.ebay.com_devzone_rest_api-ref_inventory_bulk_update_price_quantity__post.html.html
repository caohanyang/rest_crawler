Bulk Update Price and Quantity

POST /bulk_update_price_quantity

This call is used by the seller to update the total ship-to-home quantity of an inventory item and/or to update the price and/or quantity of that same inventory item in an actual live offer (aka active eBay listing). Up to 25 individual SKUs (and the active offers associated with them) may be updated with one Bulk Update Price and Quantity call.

Note: For multiple-variation listings, it is recommended that the Bulk Update Price and Quantity call be used to update price and quantity information for multiple SKUs within that multiple-variation listing instead of using separate Create or Replace Inventory Item calls to update the price and quantity for each SKU.

Request field descriptions



Input Container/Field Type Occurrence Meaning
requests array of PriceQuantity Required This container is used by the seller to update the total ship-to-home quantity of an inventory item and/or to update the price and/or quantity of that same inventory item in an actual live offer (aka active eBay listing). Up to 25 individual SKUs and associated offers can be updated under one requests container.
requests. offers array of OfferPriceQuantity Conditional Required if the seller is updating the price and/or quantity of one or more offers. This container is used to update the price and/or quantity for one or more offers associated with the corresponding sku value. It is not required if the seller is only updating the total available quantity for the inventory item through the shipToLocationAvailability container.
requests.offers
. availableQuantity 		integer Conditional Required if the seller is updating the current quantity of the offer. This integer value is used to modify the current quantity of the product (identified with the corresponding sku value) that will be available for purchase in the live offer (identified by the corresponding offerId value).
requests.offers. offerId string Conditional Required if the offers container is used. Unique identifier of the product offer. If an offers container is used to update one or more live offers, the offerId is required in order to identify the active offer to update with a modified price and/or quantity.

If an offerId value is not known, the seller can run a Get Offers call (passing in the correct SKU value as a query parameter) to retrieve offerId values for offers associated with the SKU.
requests.offers. price Amount Conditional Required if the seller is updating the current price of the offer. The dollar value set here will be the new listing price of the product (identified with the corresponding sku value) in the live offer (identified by the corresponding offerId value).
requests.offers.price. currency string Conditional Required if an amount is being expressed in request payload. A string representation of the type of currency being used.
requests.offers.price. value string Conditional Required if an amount is being expressed in request payload. A string representation of a dollar value expressed in the currency specified in the currency field.
requests
. shipToLocationAvailability 		ShipToLocationAvailability Conditional Required if the seller is updating the total available quantity for the inventory item. This container is used to specify the total quantity of the inventory item that is available for a standard purchase where the item is shipped to the buyer.
requests
.shipToLocationAvailability
. quantity 		integer Conditional Required if the original Create or Replace Inventory Item call to create the inventory item had 'ship-to-location' inventory set up, and another Create or Replace Inventory Item call is being made to update the inventory item. This integer value indicates the quantity of the inventory item that is available for standard shipping to the buyer's location.
requests. sku string Required This is the seller-defined SKU value of the inventory item that will be updated. This field is required whether the seller is updating the total quantity of the inventory item, or if the seller is updating the price and/or quantity of an active offer associated with the inventory item. Its value must match the SKU value that is associated with the offer(s) specified through the offers container.

Max length: 50.

Sample: Updating Price and Quantity

This call updates price and quantity for a SKU.

Description

This call will update the total quantity for one or more SKUs, and the price and/or quantity for one or more offers associated with those SKUs.

Input

This call will update four separate offers related to two distinct SKUs. The total ship-to-home quantity for the GP-Cam-01 product is specified in the shipToLocationAvailability container. The price and quantity for two separate offers associated with this SKU are updated. Notice that the currency types are USD and GBP , indicating that one offer is on the eBay US site and the other offer is on the eBay UK site. Similar actions are made for the product identified as GP-Cam-02 .
