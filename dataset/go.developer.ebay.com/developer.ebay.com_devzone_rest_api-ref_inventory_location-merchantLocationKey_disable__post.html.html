Disable Inventory Location

POST /location/{merchantLocationKey}/disable

Use this call to disable the inventory location specified by the { merchantLocationKey } path parameter. Sellers can not load inventory to disabled locations.

Input Output Error Codes Samples Change History

Input

See also Samples .

Resource URI (production) 

POST https://api.ebay.com/sell/inventory/v1/location/{merchantLocationKey}/disable

URI parameters

Parameter Type Required? Meaning
merchantLocationKey string Required A unique merchant-defined key (ID) for an inventory location. This value is passed in through the call URI to disable the specified inventory location.

Max length : 36


HTTP request headers

All requests made to eBay REST operations require you to provide the authorization HTTP header for authentication.

OAuth request scope

Sample: Disable a Location

Description

This example disables an existing location. Inventory items can not be associated with disabled locations. Disabled locations can later be enabled by using the Enable a Location call.

Input

Specify the location you want to disable using the inventory location's merchantLocationKey value. 

URL format. See also the non-wrapped version of this URL.

POST https://api.ebay.com/sell/inventory/v1/location/warehouse-1/disable

Output

If the call is successful, an HTTP status code of 200 and an empty response payload are returned. 

JSON format.
If successful, this operation returns an HTTP status code of "200 OK", with an empty payload.
