Create Item Promotion

POST /item_promotion

This call creates a promotion for specific inventory. You specify the inventory by passing in a list of the seller's inventory reference IDs or a list of eBay listing IDs. A seller's inventory reference ID is the identifier of a listing, also known as the SKU or custom label. You must always pass in both inventory_reference_id and inventory_reference_type . You can create a promotion as draft, scheduled, or running. The promotion ID is returned in the Location response header.

You can create promotions on a site only for active items on that site. For example, after the seller has accepted the US terms and conditions, you can create a promotion for any of your inventory that you actively listed on ebay.com. The promotion will show up only on ebay.com, even if the items can be shipped to another country, such as Austrailia. To create a promotion for domestic Australian buyers on ebay.com.au, you would need to accept the Austrailian terms and conditions and create the promotion on ebay.com.au with your active listings.

This API is supported on eBay US (United States), GB (United Kingdom), DE (Germany), AU (Australia), FR (France), IT (Italy), and ES (Spain).

To check that the promotion was created and is running, use the Get Promotions call.

For information about the Promotions manager requirements see the API requirements and restrictions section in the Marketing Overview for information about restrictions and requirements.

Request field descriptions



Input Container/Field Type Occurrence Meaning
description string Conditional The "tag" line for the offer type, such as "Save on designer shoes." It appears under the offer type text in the promotion tile that shows on the seller's all offer page and on the event page for the promotion.

Note: The offer type text is used as a teaser on the Search Results Page and the View Item Page. It's not editable by the seller and is derived from the settings in the discountRule and discountSpecificiation fields. Example of offer type text: Extra 20% off whe you buy 3+

Required unless the promotion is in DRAFT state.

Max length: 50





discountRules array of DiscountRule Conditional Container for the discount percentage or amount of the promotion. Refer to Item Promotion Rules for more information about using the discountRule and discountSpecificiation fields together to create various promotions.

Required unless the promotion is in DRAFT state.

discountRules. discountBenefit DiscountBenefit Conditional Container for the details of the buyer's discount.
discountRules.discountBenefit
. amountOffItem 		Amount Conditional The amount of discount that will be applied to an item when the promotion criteria is met. This would be used to create an 'Extra $5 off each item' offer. Every item the buyer purchases from the inventory set gets a $5 discount.

To create this promotion, you set the following discount details:
  • discountSpecification.minQuantity to 0
  • discountSpecification.forEachQuantity to 1
  • discountSpecification.numberOfDiscountedItems to 0
  • discountRules.discountBenefit.amountOffItem to 5


Note: The request can have only one of these fields: percentageOffOrder , amountOffOrder , percentageOffItem , or amountOffItem . Required: discountSpecification.numberOfDiscountedItems
