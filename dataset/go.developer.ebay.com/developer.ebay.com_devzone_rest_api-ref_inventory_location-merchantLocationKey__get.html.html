Get Inventory Location

GET /location/{merchantLocationKey}

This call retrieves the complete details of the inventory location that is specified by the { merchantLocationKey } path parameter.

Input Output Error Codes Samples Change History

Input

See also Samples .

Resource URI (production) 

GET https://api.ebay.com/sell/inventory/v1/location/{merchantLocationKey}

URI parameters

Parameter Type Required? Meaning
merchantLocationKey string Required A unique merchant-defined key (ID) for an inventory location. This value is passed in at the end of the call URI to specify the inventory location to retrieve.

Max length : 36

The geoCoordinates container will only be set if the merchant provided geographical coordinates.

Max length : 1000 locationTypes array of string Conditionally This container defines the function of the inventory location. Typically, an inventory location will serve as a store or a warehouse, but in some cases, an inventory location may be both.

Sample: Get a Location by Key

Description

This example retrieves a specific inventory location.

Input

Specify the inventory location you want to retrieve using the inventory location's merchantLocationKey value. 

URL format. See also the non-wrapped version of this URL.

GET https://api.ebay.com/sell/inventory/v1/location/store-1

Output

If the call is successful, it returns an HTTP status code of "200 OK", with a response payload that contains the details of the specified location. 

JSON format.
{
  "name": "Warehouse #1",
  "location": {
    "address": {
      "addressLine1": "2055 Hamilton Ave",
      "addressLine2": "Building 3",
      "city": "San Jose",
      "stateOrProvince": "CA",
      "postalCode": "95008",
      "country": "US"
    }
  },
  "merchantLocationStatus": "ENABLED",
  "locationTypes": [
    "WAREHOUSE"
  ],
  "merchantLocationKey": "Warehouse-1"
}
