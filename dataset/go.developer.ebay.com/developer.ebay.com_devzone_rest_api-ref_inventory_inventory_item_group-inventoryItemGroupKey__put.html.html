Create or Replace Inventory Item Group

PUT /inventory_item_group/{inventoryItemGroupKey}

This call creates a new inventory item group or updates an existing inventory item group. In the case of updating/replacing an existing inventory item group, this call does a complete replacement of the existing inventory item group record, so all fields (including the member SKUs) that make up the inventory item group are required, regardless of whether their values changed. So, when replacing/updating an inventory item group record, it is advised that the seller run a Get Inventory Item Group call for that inventory item group to see all of its current values/settings/members before attempting to update the record. And if changes are made to an inventory item group that is part of a live, multiple-variation eBay listing, these changes automatically update the eBay listing. For example, if a SKU value is removed from the inventory item group, the corresponding product variation will be removed from the eBay listing as well.

Key information that is set with this call include:

  • Seller-defined inventoryItemGroupKey value for the product group.
  • Seller-defined SKU values for each inventory item of the inventory item group. Each seller product, including products within an item inventory group, must have their own unique SKU value
  • Title and description of the inventory item group
  • Common aspects that inventory items in the qroup share
  • Product aspects that vary within each product variation
  • Links to images of the product

Request field descriptions



Input Container/Field Type Occurrence Meaning
aspects array of string Conditional This container consists of an array of aspects that are shared by all product variations within the inventory item group. Common aspects should be specified for the inventory item group, but they are not required. Common aspects for a men's t-shirt might be pattern and sleeve length. See the example below to get an idea of the syntax that is used to specify common aspects: 

"aspects": {
 "pattern": ["solid"],
 "sleeves": ["short"]
 }

Max length: 40.
description string Required The description of the inventory item group. This description should describe the product and the variations of the product that are available in the inventory item group. This field is required when creating an inventory item group. Note that this field is not the listing description field, although they may be similar. The listing description field is actually passed in the Create Offer call. In the case of a multiple-variation listing, the listingDescription should be the same for each variation of the product group.

Max length: 500000.
imageUrls array of string Required In this container, the seller will include one or more URL links to base images for the inventory item group. At least one image must be specified. The images should pertain to the aspect that is passed under the corresponding aspectsImageVariesBy container. For example, if that aspect is color, and the product is a shirt, the linked images would contain pictures of the shirts in the available colors.
inventoryItemGroupKey string Optional This is the unique identifier of the inventory item group. This identifier is created by the seller. This field is not applicable for call requests, as its value is passed in at the end of the call URI instead. This value is returned in the response of the Get Inventory Item Group call.
subtitle string Optional The listing subtitle that will be used if the product group/multiple-variation listing is listed on eBay. A subtitle is optional in a listing, and eBay will charge a small fee for this feature.

Note that the same subtitle text should be used for each inventory item that will be part of the inventory item group.

Max length: 55.
title string Required The listing title that will be used if product is listed on eBay. This field is required.

Note that the same title text should be used for each inventory item that will be part of the inventory item group.

Max length: 80.
variantSKUs array of string Required This container is used to assign individual inventory items to the inventory item group. Multiple SKU values are passed in to this container. If updating the inventory item group through a Create or Replace Inventory Item Group call, the seller will remove SKUs from this container to remove that inventory item/SKU from the inventory item group.
variesBy VariesBy Required This container is used to specify product aspects for which variations within an inventory item group vary. For example, t-shirts in an inventory item group may be available in multiple sizes and colors.
variesBy. aspectsImageVariesBy array of string Required This container will contain one product aspect where the variations of the inventory item group may vary, such as color or size.
variesBy. specifications array of Specification Required This container consists of an array of one or more product aspects where each variation differs, and values for each of those product aspects.
variesBy.specifications. name string Required This is the name of product variation aspect. For example, the aspect names in this field might be "Size" and "Color" .

Max length: 40.
variesBy.specifications. values array of string Required This is an array of values pertaining to the corresponding product variation aspect (specified in the name field). Below is a sample of how these values will appear under a specifications container: 
"specifications": [{
 "name": "Size",
 "values": ["Small",
 "Medium",
 "Large"],
 "order": 1 
 },
 { 
 "name": "Color",
 "values": ["Blue",
 "White",
 "Red"],
 "order": 2 
 }]


Max length: 50.

Sample: Creating an Inventory Item Group

This call creates an inventory item group.

Description

This call will create an inventory item group for the seller's acount.

Input

The inventoryItemGroupKey value associated with the inventory item group to create is input into the end of the call URI. In this case, the inventoryItemGroupKey value for the inventory item group is Mens_Solid_Polo_Shirts .

This particular inventory item group will be composed of men's solid polo shirts in five different colors and four different sizes. The common product aspects that each inventory item of the group shares is the pattern and the sleeve size, as specified in the aspects container. The full list of inventory items that will become part of this group is specified through the variantSKUs container. Note that inventory item records must already exist for all of these SKU values in order for the inventory item group to be created. The particular aspect where images of the shirt variations will vary is stated in the aspectsImageVariesBy container. In this particular case, an image of a shirt in each available color will be part of the listing. The available colors (and sizes) of the shirts are specified under the specifications container.
