Get All Campaigns

GET /ad_campaign


This call retrieves all your campaigns. You can filter the result set by campaign name, end date, start date, or campaign status.

Input Output Error Codes Samples Change History

Input

See also Samples .

Resource URI (production) 

GET https://api.ebay.com/sell/marketing/v1/ad_campaign?
  campaign_name=string&
  campaign_status=string&
  end_date_range=string&
  limit=string&
  offset=string&
  start_date_range=string

URI parameters

Parameter Type Required? Meaning
campaign_name string Optional Specifies the campaign name.

Note: The results might be null if other filters exclude the campaign with this name.
campaign_status string Optional Specifies the campaign status.

Note: The results might not include all the campaigns with this status if other filters exclude them.
end_date_range string Optional Specifies the range of a campaign's end date. The results will be filtered to include only campaigns with an end date that is within specified range.

Valid Format:

yyyy-MM-ddThh:mm.ss.sssZ..yyyy-MM-ddThh:mm.ss.sssZ (campaign ends within this range)
yyyy-MM-ddThh:mm.ss.sssZ.. (campaign ends on or after this date)
..yyyy-MM-ddThh:mm.ss.sssZ (campaign ends on or before this date)
2016-09-08T00:00.00.000Z..2016-09-09Thh:00.00.000Z (campaign ends on September 8, 2016)

Note: The results might not include all the campaigns ending on this date if other filters exclude them.






limit string Optional Specifies the maximum number of result sets to return.

Default: 10
Maximum: 500


offset string Optional Specifies the number of campaigns from the top of the returned items list to ignore. For example, if you set this to 5, the first campaign returned would be the 6th one in the list. This is used with the limit field to control the pagination of the output. This value is usually obtained from a previous response.

For example:
If offset is 10 and limit is 10, the call will return campaigns 11 through 20 from the list of campaigns returned.
Default: 0



start_date_range string Optional Specifies the range of a campaign's start date in which to filter the results. The results will be filtered to include only campaigns with a start date that is equal to this date or is within specified range.

Valid Format:

yyyy-MM-ddThh:mm.ss.sssZ..yyyy-MM-ddThh:mm.ss.sssZ (starts within this range)
yyyy-MM-ddThh:mm.ss.sssZ.. (campaign starts on or after this date)
..yyyy-MM-ddThh:mm.ss.sssZ (campaign starts on or before this date)
2016-09-08T00:00.00.000Z..2016-09-09Thh:00.00.000Z (campaign starts on September 8, 2016)

Note: The results might not include all the campaigns with this start date if other filters exclude them.

Default: false

Code so that your app gracefully handles any future changes to this list.

  • Each set of selection rules are ORed with each other.
  • Individual rules within a selection rule set are ANDed with each other. If a rule has a list of values, such a list of category IDs, the item must match only one of the values of the rule.

The bid percentage is a single precision value. For example, 4.1, 4.10, 5.5, 5.0, etc. are valid values and 4.44, 8.11, etc. would be invalid values.
