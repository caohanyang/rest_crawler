PricingSummary

Type that defines the fields for the costs, discounts, and seller fees and adjustments of all the line items.

Type that uses PricingSummary:

    CheckoutSessionResponse

Calls that use PricingSummary:

    /checkout_session/initiate POST , /checkout_session/{checkoutSessionId} GET , /checkout_session/{checkoutSessionId}/update_payment_info POST , /checkout_session/{checkoutSessionId}/update_quantity POST , /checkout_session/{checkoutSessionId}/update_shipping_address POST , /checkout_session/{checkoutSessionId}/update_shipping_option POST , /guest_checkout_session/initiate POST , /guest_checkout_session/{checkoutSessionId} GET , /guest_checkout_session/{checkoutSessionId}/update_payment_info POST , /guest_checkout_session/{checkoutSessionId}/update_quantity POST , /guest_checkout_session/{checkoutSessionId}/update_shipping_address POST , /guest_checkout_session/{checkoutSessionId}/update_shipping_option POST

adjustment ( Adjustment ) [0..?]
The total amount of any seller adjustments. An adjustment can be a credit or debit. This is used to catch any monetary changes to the order that are not already captured in one of the other fields.
See the Field Index to learn which calls use adjustment.

deliveryCost ( Amount ) [0..?]
The shipping cost for all the line items before any shipping discounts are applied.

Let's say there were 4 line items and the shipping cost for each line items is $5. One of the line items qualifies for free shipping and two line items qualify for a Buy 1 Get 1 offer, which would be a $6 and a $15 discount. The deliveryCost value would be 20, which is the total cost for shipping all the line items before discounts.

See the Field Index to learn which calls use deliveryCost.

deliveryDiscount ( Amount ) [0..?]
The total amount of the shipping discounts for the all line items, which include promotional shipping discounts, such as free shipping if you buy two or more.

Let's say there were 4 line items and the shipping cost for each line items is $5. One of the line items qualifies for free shipping and two line items qualify for a Buy 1 Get 1 offer, which would be a $6 and a $15 discount. The deliveryDiscounts value would be 5, which is the value of the free shipping discount.

Note:



This will always be a negative number.
See the Field Index to learn which calls use deliveryDiscount.

fee ( Amount ) [0..?]
The total amount of any fees for all the line items, such as a recycling fee.
See the Field Index to learn which calls use fee.

priceDiscount ( Amount ) [0..?]
The total amount of all the item specific discounts for all line items, such as Buy 1 Get 1 free.

Let's say there were 4 line items and the shipping cost for each line items is $5. One of the line items qualifies for free shipping and two line items qualify for a Buy 1 Get 1 offer, which would be a $6 and a $15 discount. The priceDiscount value would be 21, which is the total of the two Buy 1 Get 1 discounts.

Note:



This will always be a negative number.
See the Field Index to learn which calls use priceDiscount.

priceSubtotal ( Amount ) [0..?]
The total amount for all the line items before adding in taxes and shipping costs, or applying discounts, fees, and adjustments.
See the Field Index to learn which calls use priceSubtotal.

tax ( Amount ) [0..?]
The total amount of the taxes for all the line items.
See the Field Index to learn which calls use tax.

total ( Amount ) [0..?]
The total of the purchase order, which is:

priceSubtotal + baseDeliveryCost + tax + deliveryDiscounts + priceDiscount + adjustment .

Note: deliveryDiscounts and priceDiscount are negative numbers.



See the Field Index to learn which calls use total.
