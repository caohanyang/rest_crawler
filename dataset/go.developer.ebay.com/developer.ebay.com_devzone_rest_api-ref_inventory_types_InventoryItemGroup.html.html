InventoryItemGroup

This type is used by the base request payload of the Create or Replace Inventory Item Group call and the base response payload of the Get Inventory Item Group call.

Call that uses InventoryItemGroup:

    /inventory_item_group/{inventoryItemGroupKey} PUT

Fields

aspects ( array of string )
description ( string )
imageUrls ( array of string )
inventoryItemGroupKey ( string )
subtitle ( string )
title ( string )
variantSKUs ( array of string )
variesBy ( VariesBy )


aspects ( array of string ) [0..?]
This container consists of an array of aspects that are shared by all product variations within the inventory item group. Common aspects should be specified for the inventory item group, but they are not required. Common aspects for a men's t-shirt might be pattern and sleeve length. See the example below to get an idea of the syntax that is used to specify common aspects: 

"aspects": {
 "pattern": ["solid"],
 "sleeves": ["short"]
 }
See the Field Index to learn which calls use aspects.

description ( string ) [0..?]
The description of the inventory item group. This description should describe the product and the variations of the product that are available in the inventory item group. This field is required when creating an inventory item group. Note that this field is not the listing description field, although they may be similar. The listing description field is actually passed in the Create Offer call. In the case of a multiple-variation listing, the listingDescription should be the same for each variation of the product group.
See the Field Index to learn which calls use description.

imageUrls ( array of string ) [0..?]
In this container, the seller will include one or more URL links to base images for the inventory item group. At least one image must be specified. The images should pertain to the aspect that is passed under the corresponding aspectsImageVariesBy container. For example, if that aspect is color, and the product is a shirt, the linked images would contain pictures of the shirts in the available colors.
See the Field Index to learn which calls use imageUrls.

inventoryItemGroupKey ( string ) [0..?]
This is the unique identifier of the inventory item group. This identifier is created by the seller. This field is not applicable for call requests, as its value is passed in at the end of the call URI instead. This value is returned in the response of the Get Inventory Item Group call.
See the Field Index to learn which calls use inventoryItemGroupKey.

subtitle ( string ) [0..?]
The listing subtitle that will be used if the product group/multiple-variation listing is listed on eBay. A subtitle is optional in a listing, and eBay will charge a small fee for this feature.

Note that the same subtitle text should be used for each inventory item that will be part of the inventory item group.
See the Field Index to learn which calls use subtitle.

title ( string ) [0..?]
The listing title that will be used if product is listed on eBay. This field is required.

Note that the same title text should be used for each inventory item that will be part of the inventory item group.
See the Field Index to learn which calls use title.

variantSKUs ( array of string ) [0..?]
This container is used to assign individual inventory items to the inventory item group. Multiple SKU values are passed in to this container. If updating the inventory item group through a Create or Replace Inventory Item Group call, the seller will remove SKUs from this container to remove that inventory item/SKU from the inventory item group.
See the Field Index to learn which calls use variantSKUs.

variesBy ( VariesBy ) [0..?]
This container is used to specify product aspects for which variations within an inventory item group vary. For example, t-shirts in an inventory item group may be available in multiple sizes and colors.
See the Field Index to learn which calls use variesBy.
