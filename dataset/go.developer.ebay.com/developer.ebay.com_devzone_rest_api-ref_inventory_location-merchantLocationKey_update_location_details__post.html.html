Update Inventory Location

POST /location/{merchantLocationKey}/update_location_details

Use this call to update an existing inventory location. Specify the location you want to update using the { merchantLocationKey } path parameter.

You can update the following inventory location fields: name , phone , locationWebUrl , locationInstructions and locationAdditionalInformation . All of these fields are text-based, and whatever text is passed in for these fields in an Update Inventory Location call will replace the current values. The merchant location key, the physical location of the store, and its geo-location coordinates can not be updated with an Update Inventory Location call

Request field descriptions



Input Container/Field Type Occurrence Meaning
locationAdditionalInformation string Optional This text field is used by the merchant to provide/update additional information about an inventory location.

Max length : 256
locationInstructions string Optional This text field is used by the merchant to provide/update special pickup instructions for the store location. Although this field is optional, it is recommended that merchants provide this field to create a pleasant and easy pickup experience for In-Store Pickup and Click and Collect orders. If this field is not included in the call payload, eBay will use the default pickup instructions contained in the merchant's profile.

Max length : 1000
locationWebUrl string Optional This text field is used by the merchant to provide/update the Website address (URL) for the store location.

Max length : 512
name string Optional This text field is used by the merchant to update the name of the location. This is generally a human-friendly name as it is displayed in user interfaces.
operatingHours array of OperatingHours Optional This container is used to provide/update the regular operating hours for a store location during the days of the week. A dayOfWeekEnum field and an intervals container will be needed for each day of the week that the store location is open. Note that if operating hours are already set for an inventory location for a specific day of the week, whatever is set through an Update Interval Location call will override those hours.
operatingHours. dayOfWeekEnum string Conditional Required if you specify operatingHours for the location. A dayOfWeekEnum value is required for each day of the week that the store location has regular operating hours.

Returned if set for the store location.

Applicable values are from DayOfWeekEnum :See dayOfWeekEnum .
operatingHours. intervals array of Interval Conditional Required if you specify operatingHours for the location. This container is used to define the opening and closing times of a store's working day (defined in the dayOfWeekEnum field). An interval container is needed for each day of the week that the store location is open.

Returned if set for the store location.
operatingHours.intervals. close string Conditional Required if the intervals container is used to specify working hours or special hours for a store. The close value is actually the time that the store closes. Local time (in Military format) is used. So, if a store closed at 8 PM local time, the close time would look like the following: 20:00:00 .

Returned if set for the store location.
operatingHours.intervals. open string Conditional Required if the intervals container is used to specify working hours or special hours for a store. The open value is actually the time that the store opens. Local time (in Military format) is used. So, if a store opened at 9 AM local time, the open time would look like the following: 09:00:00 .

Returned if set for the store location.
phone string Optional This text field is used by the merchant to provide/update the phone number for the inventory location.

Max length : 36
specialHours array of SpecialHours Optional This container is used to provide/update the special operating hours for a store location on a specific date. The special hours specified for the specific date will override the normal operating hours for that particular day of the week. If special hours have already been set up for an inventory location, specifying special hours through an Update Interval Location call will only add to the list, unless the date(s) used are the same special date(s) already set up, in which case, the special hours set up through the Update Interval Location call will override the existing special hours.
specialHours. date string Conditional Required if specialHours container is used. This is a specific date for which this set of special hours apply. The date is actually expressed as a timestamp specified in ISO 8601 format, which uses the 24-hour Coordinated Universal Time (UTC) clock. The following examples show (1) the format of the time-stamp, and (2) an example time value in ISO 8601 format:

Format: YYYY-MM-DDTHH:MM:SS.SSSZ
Example: 2016-10-19T00:09:00.000Z

Returned if set for the store location.
specialHours. intervals array of Interval Conditional Required if you specify specialHours for the location. This container is used to define the opening and closing times of the store location for the date specified in the date field. These hours will override the standard operating hours for that particular day of the week.

Returned if set for the store location.
specialHours.intervals. close string Conditional Required if the intervals container is used to specify working hours or special hours for a store. The close value is actually the time that the store closes. Local time (in Military format) is used. So, if a store closed at 8 PM local time, the close time would look like the following: 20:00:00 .

Returned if set for the store location.
specialHours.intervals. open string Conditional Required if the intervals container is used to specify working hours or special hours for a store. The open value is actually the time that the store opens. Local time (in Military format) is used. So, if a store opened at 9 AM local time, the open time would look like the following: 09:00:00 .

Returned if set for the store location.

Sample: Update an Inventory Location

Description

This example updates an existing inventory location.

Input

Specify the inventory location you want to update using the merchantLocationKey path parameter. In the following sample, the location's name, phone number, and Web site are being updated, as well as new information and instructions for the location.

    "locationAdditionalInformation": "Available for drop-off on Mondays only.",
    "locationWebUrl": "http://www.example.com/warehouse-1",

}

If the call is successful, eBay returns an HTTP status code of 204 No Content , with an empty payload. 

JSON format.
Upon success, this call returns an HTTP status code of "204 No Content" and an empty response payload.
