InventoryLocationFull

This type is used by the Create Inventory Location call to provide details on the inventory location, including the location's name, physical address, operating hours, special hours, phone number and other details of an inventory location.

Call that uses InventoryLocationFull:

    /location/{merchantLocationKey} POST

Fields

location ( Location )
locationAdditionalInformation ( string )
locationInstructions ( string )
locationTypes ( array of StoreTypeEnum )
locationWebUrl ( string )
merchantLocationStatus ( StatusEnum )
name ( string )
operatingHours ( array of OperatingHours )
phone ( string )
specialHours ( array of SpecialHours )


location ( Location ) [0..?]
This container is used to set the physical address of an inventory location and geographical coordinates (optional).
See the Field Index to learn which calls use location.

locationAdditionalInformation ( string ) [0..?]
This text field is used by the merchant to provide additional information about an inventory location.

Max length : 256
See the Field Index to learn which calls use locationAdditionalInformation.

locationInstructions ( string ) [0..?]
This text field is used by the merchant to provide special pickup instructions for the store location. Although this field is optional, it is recommended that merchants provide this field to create a pleasant and easy pickup experience for In-Store Pickup and Click and Collect orders. If this field is not included in the call payload, eBay will use the default pickup instructions contained in the merchant's profile.

Note: This field is not yet applicable since establishing In-Store Pickup or Click and Collect inventory is not supported with the the initial release of the Inventory API. Max length : 1000
See the Field Index to learn which calls use locationInstructions.

locationTypes ( array of StoreTypeEnum ) [0..?]
This container is used to define the function of the inventory location. Typically, an inventory location will serve as a store or a warehouse, but in some cases, an inventory location may be both.

If this container is omitted, the location type of the inventory location will default to WAREHOUSE

Default : WAREHOUSE
See the Field Index to learn which calls use locationTypes.

locationWebUrl ( string ) [0..?]
This text field is used by the merchant to provide the Website address (URL) for the inventory location.

Max length : 512
See the Field Index to learn which calls use locationWebUrl.

merchantLocationStatus ( StatusEnum ) [0..?]
This field is used to indicate whether the inventory location will be enabled (inventory can be loaded to location) or disabled (inventory can not be loaded to location). If this field is omitted, a successful Create Inventory Location call will automatically enable the inventory location. A merchant may want to create a new inventory location but leave it as disabled if the inventory location is not yet ready for active inventory. Once the inventory location is ready, the merchant could use the Enable Inventory Location call to enable the inventory location in disabled status.

Default : ENABLED
See the Field Index to learn which calls use merchantLocationStatus.

name ( string ) [0..?]
The name of the inventory location. This name should be a human-friendly name as it will be displayed in user interfaces.

Max length : 1000
See the Field Index to learn which calls use name.

operatingHours ( array of OperatingHours ) [0..?]
Required if the inventory location is a store that will handle In-Store Pickup or Click and Collect orders. This container is used to express the regular operating hours for a store location during the days of the week. A dayOfWeekEnum field and an intervals container will be needed for each day of the week that the store location is open.

Note: This container is not yet applicable since establishing In-Store Pickup or Click and Collect inventory is not supported with the the initial release of the Inventory API.
See the Field Index to learn which calls use operatingHours.

phone ( string ) [0..?]
Required if the inventory location is a store that will handle In-Store Pickup or Click and Collect orders. The phone number for the store location.

Max length : 36
See the Field Index to learn which calls use phone.

specialHours ( array of SpecialHours ) [0..?]
This container is used to express the special operating hours for store location on a specific date. The special hours specified for the specific date will override the normal operating hours for that particular day of the week.

Note: This container is not yet applicable since establishing In-Store Pickup or Click and Collect inventory is not supported with the the initial release of the Inventory API.
See the Field Index to learn which calls use specialHours.
